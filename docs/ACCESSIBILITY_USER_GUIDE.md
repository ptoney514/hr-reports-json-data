## Accessibility Overview

### Commitment to Accessibility

The HR Reports System is designed to be accessible to all users, regardless of abilities or assistive technologies used. Our system meets **WCAG 2.1 AA standards** and provides comprehensive accessibility features to ensure an inclusive experience for everyone.

### Accessibility Standards

Our system complies with:
- **WCAG 2.1 Level AA** - Web Content Accessibility Guidelines
- **Section 508** - US Federal accessibility requirements
- **ADA** - Americans with Disabilities Act guidelines
- **EN 301 549** - European accessibility standard

### Who This Guide Helps

This guide is designed for users who:
- Use screen readers or other assistive technologies
- Navigate primarily with keyboard input
- Have visual impairments or color vision differences
- Have motor disabilities affecting mouse/touch use
- Have cognitive differences affecting information processing
- Simply prefer accessible design for better usability

## Getting Started with Accessibility

### Accessing Accessibility Settings

#### Step 1: Locate Accessibility Controls
1. **Open the HR Reports System** in your web browser
2. **Look for the "Accessibility" button** in the main navigation menu
3. **Click on "Accessibility"** or press **Alt+A** to open the accessibility panel

#### Step 2: Review Available Options
The accessibility panel includes these features:
- **High Contrast Mode** - Enhanced color contrast
- **Large Text Mode** - Increased font sizes
- **Reduced Motion** - Minimized animations
- **Screen Reader Mode** - Optimized for assistive technologies
- **Enhanced Focus** - More visible focus indicators
- **Keyboard Shortcuts** - Display available shortcuts

#### Step 3: Configure Your Preferences
1. **Toggle switches** to enable/disable features
2. **Settings are saved automatically** for future visits
3. **Test each feature** to ensure it meets your needs
4. **Adjust as needed** for optimal experience

### Quick Setup Guide

#### For Screen Reader Users
1. **Enable "Screen Reader Mode"** for optimized experience
2. **Enable "Enhanced Focus"** for better navigation
3. **Enable "Keyboard Shortcuts"** for efficiency

#### For Low Vision Users
1. **Enable "High Contrast Mode"** for better visibility
2. **Enable "Large Text Mode"** for easier reading
3. **Adjust browser zoom** to 150-200% if needed

#### For Motor Impairment Users
1. **Enable "Enhanced Focus"** for easier targeting
2. **Enable "Keyboard Shortcuts"** for mouse alternatives
3. **Enable "Reduced Motion"** to prevent distractions

## Accessibility Features

### High Contrast Mode

#### What It Does
- **Increases color contrast** to 7:1 ratio or higher
- **Uses dark text on light backgrounds** for better readability
- **Maintains chart readability** with accessible color palettes
- **Improves focus indicators** and interactive element visibility

#### How to Use
1. **Click "High Contrast Mode"** toggle in accessibility panel
2. **Page immediately updates** with high contrast colors
3. **All charts and graphs** use accessible color schemes
4. **Print outputs** automatically use high contrast

#### Best Practices
- **Combine with Large Text Mode** for maximum readability
- **Use with browser zoom** for optimal viewing
- **Test with your preferred color settings**

### Large Text Mode

#### What It Does
- **Increases font sizes** by 125% throughout the application
- **Scales interface elements** proportionally
- **Maintains layout integrity** while improving readability
- **Affects all text** including charts, tables, and buttons

#### How to Use
1. **Enable "Large Text Mode"** in accessibility panel
2. **All text sizes increase** automatically
3. **Layouts adjust** to accommodate larger text
4. **Combine with browser zoom** for further enlargement

#### Font Size Reference
- **Normal Mode**: 14px base font size
- **Large Text Mode**: 17.5px base font size
- **With 150% browser zoom**: 26.25px effective size

### Reduced Motion

#### What It Does
- **Minimizes animations** and transitions
- **Respects prefers-reduced-motion** system setting
- **Maintains functionality** while reducing visual distraction
- **Improves focus** for motion-sensitive users

#### How to Use
1. **Enable "Reduced Motion"** in accessibility panel
2. **Animations become subtle** or disappear entirely
3. **Chart transitions** become instant or simplified
4. **Loading indicators** use static alternatives

### Screen Reader Mode

#### What It Does
- **Adds comprehensive ARIA labels** for all elements
- **Provides detailed descriptions** for charts and graphs
- **Includes data tables** as alternatives to visual charts
- **Optimizes heading structure** for navigation

#### How to Use
1. **Enable "Screen Reader Mode"** before using assistive technology
2. **Charts include detailed descriptions** spoken by screen readers
3. **Data tables provide** all visual information in accessible format
4. **Navigation becomes more descriptive** and informative

### Enhanced Focus

#### What It Does
- **Makes focus indicators** more visible and prominent
- **Increases focus ring size** and contrast
- **Improves keyboard navigation** visual feedback
- **Adds focus sounds** (optional) for audio feedback

#### How to Use
1. **Enable "Enhanced Focus"** in accessibility panel
2. **Tab through interface** to see enhanced focus indicators
3. **Focus rings are** thicker and higher contrast
4. **Audio feedback** provides additional confirmation (if enabled)

### Keyboard Shortcuts

#### What It Does
- **Displays available shortcuts** in an overlay
- **Provides quick access** to common functions
- **Enables efficient navigation** without mouse
- **Includes dashboard-specific shortcuts**

#### How to Use
1. **Enable "Keyboard Shortcuts"** to see available options
2. **Press "?" key** anytime to show shortcut overlay
3. **Use shortcuts** for faster navigation and actions
4. **Customize shortcuts** based on your workflow

## Keyboard Navigation

### Navigation Basics

#### Primary Navigation Keys
- **Tab** - Move forward through interactive elements
- **Shift+Tab** - Move backward through interactive elements
- **Enter** - Activate buttons, links, and selected items
- **Space** - Toggle checkboxes, expand dropdowns
- **Escape** - Close dialogs, cancel operations, return to previous level

#### Arrow Key Navigation
- **Arrow Keys** - Navigate within menus, charts, and data tables
- **Home** - Move to first item in a list or beginning of line
- **End** - Move to last item in a list or end of line
- **Page Up/Page Down** - Scroll through long lists or content

### Dashboard Navigation

#### Moving Between Dashboards
1. **Tab to navigation menu** (or press Alt+N)
2. **Use arrow keys** to highlight desired dashboard
3. **Press Enter** to navigate to selected dashboard
4. **Press Alt+H** to return to Dashboard Index

#### Within Dashboard Navigation
1. **Tab through sections** in logical order:
   - Filter controls
   - Summary cards
   - Charts and visualizations
   - Data tables
   - Export buttons

2. **Use landmark navigation** (if using screen reader):
   - **H** - Navigate by headings
   - **L** - Navigate by landmarks
   - **T** - Navigate by tables

### Chart Navigation

#### Accessing Chart Data
1. **Tab to chart** to focus on it
2. **Press Enter** to enter chart navigation mode
3. **Use arrow keys** to move between data points
4. **Press Tab** to move between chart elements (legend, axes)
5. **Press Escape** to exit chart navigation

#### Chart Interaction
- **Left/Right arrows** - Move between data points or categories
- **Up/Down arrows** - Move between data series
- **Enter** - Select/deselect data series
- **Space** - Toggle data series visibility
- **Home/End** - Jump to first/last data point

### Filter Navigation

#### Using Filter Controls
1. **Tab to filter section** (or press Alt+F)
2. **Arrow keys** to move between filter options
3. **Space** to open dropdown menus
4. **Type** to search within filter options
5. **Enter** to apply selected filters

#### Filter Shortcuts
- **Alt+F** - Focus on filters
- **Alt+C** - Clear all filters
- **Alt+A** - Apply current filter selection

### Table Navigation

#### Data Table Navigation
1. **Tab to table** to focus on it
2. **Arrow keys** to move between cells
3. **Home/End** - Move to beginning/end of row
4. **Ctrl+Home/End** - Move to beginning/end of table
5. **Page Up/Down** - Scroll through large tables

#### Table Features
- **Enter on header** - Sort by column
- **Space on row** - Select row (if selectable)
- **Ctrl+A** - Select all rows (if applicable)

## Screen Reader Support

### Supported Screen Readers

The HR Reports System is tested and compatible with:
- **NVDA** (Windows) - Free, open-source screen reader
- **JAWS** (Windows) - Popular commercial screen reader
- **VoiceOver** (macOS/iOS) - Built-in Apple screen reader
- **TalkBack** (Android) - Built-in Android screen reader
- **Narrator** (Windows) - Built-in Windows screen reader

### Screen Reader Optimization

#### ARIA Implementation
- **Comprehensive ARIA labels** describe all interactive elements
- **ARIA descriptions** provide additional context where needed
- **Live regions** announce dynamic content changes
- **ARIA landmarks** help with page structure navigation

#### Content Structure
- **Logical heading hierarchy** (H1, H2, H3) for easy navigation
- **Descriptive link text** that makes sense out of context
- **Table headers** properly associated with data cells
- **Form labels** clearly associated with input fields

### Chart Accessibility

#### Chart Descriptions
When Screen Reader Mode is enabled, charts include:
- **Chart type and purpose** (e.g., "Bar chart showing turnover by department")
- **Data summary** (e.g., "Engineering has highest turnover at 15%")
- **Trend information** (e.g., "Overall trend is decreasing over time")
- **Key insights** (e.g., "Q2 shows significant improvement")

#### Data Tables for Charts
Every chart has an associated data table that includes:
- **All chart data** in accessible table format
- **Proper table headers** and cell associations
- **Sortable columns** for data exploration
- **Summary statistics** and calculated values

#### Chart Navigation Commands
- **Enter chart** - Press Enter when chart is focused
- **Navigate data points** - Use arrow keys
- **Hear data values** - Data announced as you navigate
- **Access data table** - Press T when in chart
- **Exit chart** - Press Escape

### Best Practices for Screen Reader Users

#### Efficient Navigation
1. **Use heading navigation** (H key) to quickly move between sections
2. **Use landmark navigation** (D key) to jump between page regions
3. **Use table navigation** (T key) to find data tables
4. **Use link navigation** (K key) to find actionable items

#### Getting Oriented
1. **Listen to page title** and main heading first
2. **Review available landmarks** using landmark list
3. **Scan headings** to understand page structure
4. **Check for skip links** to jump to main content

#### Working with Data
1. **Find data tables** using table navigation
2. **Review table structure** using table reading commands
3. **Use chart descriptions** before accessing data tables
4. **Export data** for offline analysis if needed

## Visual Accessibility

### Color Vision Support

#### Color-Blind Friendly Design
- **Charts use patterns and textures** in addition to color
- **High contrast ratios** ensure visibility for all color vision types
- **Colorblind-safe palettes** tested with color vision simulators
- **Text labels** accompany color-coded information

#### Color Combinations
The system uses these accessible color combinations:
- **Primary blue** (#0066CC) with white text (8.2:1 contrast)
- **Success green** (#00AA44) with white text (4.7:1 contrast)
- **Warning orange** (#FF8800) with black text (4.5:1 contrast)
- **Error red** (#CC0000) with white text (7.4:1 contrast)

### Low Vision Support

#### Text Scaling
- **Responsive to browser zoom** up to 400% without loss of functionality
- **Large Text Mode** provides 125% increase in font sizes
- **Relative units** ensure proportional scaling
- **Maintains readability** at all zoom levels

#### Layout Adaptation
- **Flexible layouts** that reflow with text size changes
- **Minimum touch targets** of 44x44 pixels for easier selection
- **Adequate spacing** between interactive elements
- **Clear visual hierarchy** with appropriate contrast

### Contrast Requirements

#### Text Contrast
- **Normal text**: Minimum 4.5:1 contrast ratio
- **Large text**: Minimum 3:1 contrast ratio
- **Interactive elements**: Minimum 4.5:1 contrast ratio
- **High Contrast Mode**: 7:1 or higher contrast ratios

#### Non-Text Contrast
- **Charts and graphs**: 3:1 contrast for visual elements
- **Interactive element borders**: 3:1 contrast with adjacent colors
- **Focus indicators**: 3:1 contrast with background

## Troubleshooting Accessibility Issues

### Common Issues and Solutions

#### Screen Reader Not Announcing Content

**Problem**: Screen reader silent on certain elements
**Solutions**:
1. **Enable Screen Reader Mode** in accessibility panel
2. **Refresh the page** after enabling
3. **Update screen reader** to latest version
4. **Try different browsing mode** (browse vs. focus mode)
5. **Check browser compatibility** with your screen reader

#### Keyboard Navigation Not Working

**Problem**: Can't navigate with keyboard alone
**Solutions**:
1. **Press Tab repeatedly** to ensure you're in the tab sequence
2. **Check if focus is trapped** in a dialog or menu
3. **Press Escape** to exit modal dialogs
4. **Refresh the page** if navigation seems broken
5. **Try different browser** for compatibility

#### High Contrast Mode Issues

**Problem**: High contrast colors are hard to see
**Solutions**:
1. **Adjust browser zoom** to 150-200%
2. **Enable Large Text Mode** in addition to high contrast
3. **Check monitor brightness** and contrast settings
4. **Try different browser** for better color rendering
5. **Contact support** if colors are still problematic

#### Charts Not Accessible

**Problem**: Can't access chart information
**Solutions**:
1. **Enable Screen Reader Mode** for better chart descriptions
2. **Tab to chart and press Enter** to enter chart navigation
3. **Press T** to access data table version of chart
4. **Use export function** to get data in accessible format
5. **Try different chart type** from filter options

#### Focus Indicators Not Visible

**Problem**: Can't see where keyboard focus is located
**Solutions**:
1. **Enable Enhanced Focus** in accessibility panel
2. **Check browser zoom level** - focus may be off-screen
3. **Adjust monitor contrast** settings
4. **Try High Contrast Mode** for better visibility
5. **Use different browser** for better focus indication

### Browser-Specific Issues

#### Chrome
- **Enable accessibility features** in chrome://settings/accessibility
- **Check extension conflicts** by testing in incognito mode
- **Update to latest version** for best compatibility

#### Firefox
- **Check accessibility.* preferences** in about:config
- **Disable problematic extensions** temporarily
- **Try Firefox accessibility developer tools**

#### Safari
- **Enable VoiceOver** in System Preferences
- **Check Safari accessibility settings**
- **Try with different VoiceOver settings**

#### Edge
- **Enable Narrator** in Windows settings
- **Check Edge accessibility features**
- **Try with different browser engine** if available
