## Documentation Standards

### Markdown Files
- Use GitHub-flavored Markdown
- Include table of contents for files > 200 lines
- Use relative links for internal references
- Keep lines under 120 characters when possible

### Updates
- Update `Last Updated` date when making changes
- Add version history for major revisions
- Archive old versions instead of deleting

### Structure
- Use clear heading hierarchy (H1 > H2 > H3)
- Include code examples in triple backticks
- Add diagrams/screenshots when helpful
- Link to related documentation
